# CHG-018 상품·서비스 카테고리 정합화

담당자 7의 `F-COM-003`, `REQ-COM-003` 카테고리 제공 계약을 실제 개발 DB에 반영하는 증분 SQL이다.

## 최종 활성 카테고리

- 상품: 전자기기, 생활·가구, 패션·의류, 도서·음반, 취미, 스포츠·레저, 유아·아동, 뷰티·미용, 식품, 기타
- 서비스: 이사, 청소, 레슨, 설치·수리, 인테리어
- 상품은 모두 `CAT_PRF_YN=N`, 서비스는 모두 `CAT_PRF_YN=Y`다.
- 승인 방식은 MVP 정본에 따라 모두 `CATC0004(서류심사)`다.

## 티켓 제외 원칙

티켓은 재판매·부정 거래 위험 때문에 활성 상품 카테고리에서 완전히 제외한다. 기존 `티켓/취미`의 FK 참조가 없으면 행을 삭제하고, 다른 환경에서 참조가 있으면 물리 삭제 대신 사용 중지한다. `취미`는 티켓과 분리한 새 행으로 관리한다.

## 실행 방법

1. `CATEGORY`와 참조 테이블을 백업하거나 스냅샷을 확보한다.
2. [20260717_CHG-018_category_catalog.sql](./20260717_CHG-018_category_catalog.sql)을 MySQL 8의 대상 개발 DB에서 실행한다.
3. 마지막 조회 결과가 상품 10개, 서비스 5개인지 확인한다.
4. `GET /api/categories?domainCd=CATC0001`과 `CATC0002` 응답을 확인한다.

전체 `08_DB_기초데이터`를 기존 DB에 다시 실행하면 PK 중복이 발생하므로 사용하지 않는다. 이 증분 SQL만 실행한다.

## 애플리케이션 계약

- 공개 목록: `GET /api/categories?domainCd={CATC0001|CATC0002}`
- 관리자 관리: `/api/admin/categories/**` (`ROLE_ADMIN` 필수)
- 물리 삭제 API는 제공하지 않으며 일반 운영 변경은 사용 중지로 처리한다.
